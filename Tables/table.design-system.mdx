# ðŸ“’ Developer style guide for Tables
Table is one attractive component. That are mainly used for the representing data in field in relational manner. It is helpful because it helps to group and orgainise data in very efficient manner.

In the web-pages we use the table in directly and indirectly manner to fetch data from backend and update in table componenet. There are different version of table is available as per the user need it can be used to make your page more dynamic.

## General must known design system guideline :

### Color ðŸŽ¨

|  | Border Dark | Border Light | Backgroung Dark | Backgroung Light |
| --- | --- | --- | --- | --- |
| Code in Tailwind | border-slate-800 | border-none | bg-[#252B42] / bg-[#202537] | bg-gray-100 / bg-white |

Color of `bg` is generally used as shade of 100 / 700 

Color of `text` will depend on the `bg` color
| Background | Dark | Light |
| --- | --- | --- |
| Text | Light | Dark |
| Code in Tailwind | text-white | text-gray-900  |

`Note` : Color of the text `[color]` will depend on the color of the background

### Font

For Table font is : **`font-medium font-light`**

For Table head font is : **`font-bold`**


### Transition ( `transition-all` )

whenever there is a hover effect, one must use `transition-all` class with it.


### Hover Color

For all table row the hover color in hoverable table is **`hover:bg-blue-600`**.

### Border Radius ðŸ”²

Border radius of each table is `none`. Donot need to give any border radius class as well. 

### **Paddings :**

Generally for the padding we use. `px-6 py-4`

### Normal table guidelines :

|  | Padding | Margin | Text Size |
| --- | --- | --- | --- |
| Small | px-6 | mx-6 | text-sm |
| Large | px-8 | mx-8 | text-lg |
