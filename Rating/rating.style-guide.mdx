# ðŸ“’ Developer style guide for Rating 

Allow users to easily view and set ratings that reflect degrees of satisfaction with content and services. The rating control has several great features that provide flexibility and customization. To create rating, add role rating with attribute data-role to input element.

Simply define the how much users are satisfied with provided service and product. Later on these data help them improve their service based on data collected. Very common case you often come accross is about product from e-commerce website

## General must known design system guideline :

### Color ðŸŽ¨

| Color | Yellow | Gray | Dark-Gray |Blue|
| --- | --- | --- | --- |--- |
| Code in Tailwind | text-yellow-400 | text-gray-300 | text-gray-500 |text-blue-500|

Color of `text` will depend on the `bg` color
| Background | Dark | Light |
| --- | --- | --- |
| Text | Light | Dark |
| Code in Tailwind | bg-default | bg-gray-700 |

`Note` : Color of the text `[color]` will depend on the color of the background

### Font

For all ratings font is : **`font-medium`**

### Hover Scale

For emoji rating type while hovering on any emoji it scale as **`hover:-translate-y-1 `**

### SVG And Path
SVG stands for Scalable Vector Graphic. It can be used to make graphics and animations like in HTML canvas.  Path is always used within the svg tag to set the actual coordinate.
Wihin the tag we give the some propety as what with be the co-ordinate, fill color, outer stroke, and some styling within class attribute.

## Icon guidelines:

Mostly all the icons are taken from â†’ [Hero Icons](https://heroicons.com/)

`Icon-text-gap` :  We follow a convention of equal margin between the text and the icon as padding in y  axis `py`. and always the margin is given from the side of icon.

|  | Width | Type |
| --- | --- | --- |
| Small | w-4 & h-4 | Hero Icons mini |
| Medium | w-5 & h-5 | Hero Icons mini |
| Large | w-6 & h-6 | Hero Icons filled |
