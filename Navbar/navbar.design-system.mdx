# Navbar Design System ‚≠ê



## General convention to follow :
The componet should contain insdie a `<nav></ nav>` tag.

### **Border-radius** :

Border radius of each card is `none`. Donot need to give any border radius class as well. 

### **Paddings :**

Generally for the outermost padding we use. `px-6 py-4`

### Font :

|  | heading / logo | Normal text / Buttons or anchor tag | sub text | badge type |
| --- | --- | --- | --- | --- |
| size | text-2xl | text-lg | text-sm | text-sm / text-md |
| color | text-gray-800 | text-gray-800 / text-gray-600 ( according to the design) | text-gray-600 | text-gray-400 |
| font | semi-bold | semibold / normal | normal | normal |

### Width & heights :

The width of the component totally depends on the design.

But we usually use these 3 breakpoints üîΩ

- `md` ‚Üí This will be default one. Breaks form the mobile view to desktop view
- `sm` ‚Üí This is used ocassionally for complex Navbars for tablet views.

The height of the components depends on the amount of content on it. We need to make it fluid. **Giving a fixed height is not recommended at all.** 

---

### Anchor tag :

- **color** : `text-blue-600`
- **underline** : On hover there is an underline ‚Üí `hover:underline`
- **underline offset** : `underline-offset-4`

### Buttons :

**We will be mostly using the anchor tags for all navigation.** 


We need to use the already built buttons for this. **[Buttons](https://github.com/Clueless-Community/seamless-ui/tree/main/Buttons)** üîó¬†. Don‚Äôt use any buttons outside this.

### Icons :

Icons must be used from **[Heroicons](https://heroicons.com/)** üñáÔ∏è

**Note :** You need to use a solid 24X24 SVG form heroicons.

### Images & Names:

For the name you must use the name : `SeamLess UI`


For the logo you must use : [`https://i.ibb.co/jGMgp7r/seamless-ui.png`](https://i.ibb.co/jGMgp7r/seamless-ui.png)
